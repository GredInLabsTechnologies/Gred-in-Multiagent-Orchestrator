## Issues

### [BLOCKER] Draft/run execution halts and cannot resume
- **Probe**: A
- **Command/Tool**: `gimo_create_draft` -> `gimo_approve_draft(auto_run=true)` -> `gimo_get_task_status` -> `gimo_resolve_handover(decision="approve")`
- **Expected**: The engineering-calculator task should progress from draft to executable run and either finish or resume after explicit handover approval.
- **Actual**: The run transitioned `draft -> approved -> running -> HUMAN_APPROVAL_REQUIRED`; `gimo_resolve_handover` then returned `405 Method Not Allowed`.
- **Suspicion**: The MCP resume path is wired to a missing or mismatched backend workflow endpoint.

### [CRITICAL] OpenAI worker spawn bypasses provider readiness checks
- **Probe**: B
- **Command/Tool**: `python gimo.py providers test openai`; `gimo_spawn_subagent(provider="openai", model="auto")`; `gimo_list_agents`
- **Expected**: An unauthenticated/unreachable provider should block worker creation or fail loudly before the worker is registered.
- **Actual**: CLI reported `Provider 'openai' endpoint is unreachable` and `Auth: not authenticated`; MCP still returned `Spawned: 5d53c83c-ff8a-4b49-8d00-93429967b43b`.
- **Suspicion**: Provider validation is deferred or absent on the spawn registration path.

### [GAP] Cost, trust, and traces remain empty after apparent work
- **Probe**: A / B / C
- **Command/Tool**: `gimo_chat`, `gimo_spawn_subagent`, `gimo_get_budget_status`, `python gimo.py observe metrics`, `python gimo.py trust status`, `python gimo.py observe traces`
- **Expected**: Successful chat/spawn activity should leave at least one observable budget, token, trust, or trace signal.
- **Actual**: Budget remained zero, metrics stayed at zero tokens/cost, trust reported `No trust data yet`, traces reported `No traces recorded yet`.
- **Suspicion**: Local execution is happening through a non-metered/non-traced path, or the instrumentation hook is not firing for these flows.

### [GAP] Proof chain verifies an empty chain as valid
- **Probe**: A
- **Command/Tool**: `gimo_verify_proof_chain(thread_id="thread_2e0da91c")`
- **Expected**: A thread with executed agent turns should either produce non-zero proof length or return an explicit "no proof entries" failure.
- **Actual**: Returned `valid=true` with `length=0`.
- **Suspicion**: The verifier currently treats absence of entries as vacuous validity.

### [INCONSISTENCY] Active-run status disagrees with observability metrics
- **Probe**: C / D
- **Command/Tool**: `python gimo.py status`; `python gimo.py observe metrics`
- **Expected**: A run shown as active in status should also be reflected in active-run metrics.
- **Actual**: `status` showed `Active run: r_1775616546449_4514ef (HUMAN_APPROVAL_REQUIRED)` while metrics returned `active_runs=0`.
- **Suspicion**: Metrics exclude approval-halted runs or are lagging behind authoritative status.

### [FRICTION] Launcher split is operator-confusing
- **Probe**: C
- **Command/Tool**: `.\\gimo.cmd --help`; `.\\gimo.cmd status`; `python gimo.py --help`
- **Expected**: The official launcher should either expose the operator commands directly or clearly hand off to the richer CLI.
- **Actual**: `gimo.cmd` only exposed a small wrapper surface; `status` was unknown there but valid in `python gimo.py`.
- **Suspicion**: Wrapper and Python CLI have diverged into two user-facing contracts.

### [FRICTION] `watch --timeout 5` did not honor the operator timeout
- **Probe**: C
- **Command/Tool**: `python gimo.py watch --timeout 5`
- **Expected**: The command should return in about 5 seconds.
- **Actual**: The shell killed it after ~34 seconds.
- **Suspicion**: The CLI timeout flag is not wired to stream cancellation or is being ignored by the backend stream loop.

### [GAP] Raw HTTP feature matrix is auth-blocked for a local black-box operator
- **Probe**: D
- **Command/Tool**: `GET /ops/capabilities`, `GET /ops/graph`, `POST /ops/threads`, and related `/ops/*` calls
- **Expected**: Either a documented local auth bootstrap exists for direct API probing, or the local operator can obtain one easily within the same black-box path.
- **Actual**: All tested `/ops/*` calls returned `401 Unauthorized`; only `/health`, `/ops/health/info`, and `/auth/check` were reachable anonymously.
- **Suspicion**: Direct API access requires a separate auth handshake that is not surfaced by this Phase 1 path.
