## Full Issue List

### [BLOCKER] #1 — MCP Bridge Crashes and Does Not Recover

- **Probe**: A
- **Command/Tool**: `gimo_verify_proof_chain` → subsequent tools
- **Expected**: MCP bridge handles errors gracefully and continues serving
- **Actual**: After `verify_proof_chain` throws `ImportError: No module named 'tools.gimo_server.services.storage.storage_service'`, the MCP bridge process dies. All subsequent MCP calls fail with "Server gimo unavailable". No auto-recovery.
- **Suspicion**: Unhandled exception in MCP bridge's request handler propagates to process level. Missing try/except around tool dispatch.

### [BLOCKER] #2 — verify_proof_chain ImportError

- **Probe**: A
- **Command/Tool**: `gimo_verify_proof_chain(thread_id="thread_716d5f4f")`
- **Expected**: Returns proof chain validation result
- **Actual**: `{"valid": false, "length": 0, "error": "No module named 'tools.gimo_server.services.storage.storage_service'"}`
- **Suspicion**: `storage_service.py` was renamed/moved/deleted but the import in proof chain code was not updated.

### [BLOCKER] #3 — Run Execution Never Starts

- **Probe**: B, D
- **Command/Tool**: `POST /ops/drafts/{id}/approve?auto_run=true`
- **Expected**: Run progresses through stages (planning → executing → done), LLM calls made
- **Actual**: Run stays at `status: running, stage: null` with only 2 log entries ("Run created", "Execution started"). No LLM calls made. Eventually goes to `error` on server restart.
- **Suspicion**: Run worker fails to pick up the run. Either the worker is not started, or the execution path from approval to worker dispatch is broken.

### [BLOCKER] #4 — CLI plan and chat Commands Produce No Output

- **Probe**: C
- **Command/Tool**: `gimo plan "..." -w <path> --no-confirm`, `gimo chat -m "..." -w <path>`
- **Expected**: Plan generation or chat response from provider
- **Actual**: Commands hang with zero output until timeout. No error message.
- **Suspicion**: CLI calls the server which calls the provider, but the provider call hangs or the CLI doesn't read the streaming response.
